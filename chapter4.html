Base case, recursive case en call stack

Samenvatting

Recursie betekent dat een functie zichzelf aanroept om een kleinere versie van hetzelfde probleem op te lossen.

Elke recursieve functie moet minstens één base case hebben. Zonder base case blijf je calls opstapelen tot stack overflow.

Recursie is geen aparte strategie, maar een implementatietechniek die vaak gebruikt wordt bij backtracking, divide & conquer en sommige DP-oplossingen.

Wat moet je begrijpen?

  • Base case: stopconditie.
  • Recursive case: roept zichzelf aan met kleiner probleem.
  • Call stack: onthoudt actieve functieaanroepen.
  • Stack overflow: te veel recursieve calls zonder stop.

Big O & complexiteit

  • Factorial recursion: O(n) time, O(n) stack space.
  • Naive Fibonacci: O(2^n) time, O(n) stack space.
  • Tail of simpele recursion met één call: vaak O(n).

C# voorbeeld

static int Factorial(int n)
{
    if (n <= 1) return 1;
    return n * Factorial(n - 1);
}

Typische valkuilen

  • Base case vergeten.
  • Probleem niet kleiner maken.
  • Niet beseffen dat elke call geheugen gebruikt.
  • Naïeve Fibonacci als efficiënt beschouwen.
