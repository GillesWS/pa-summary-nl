Algorithms Fundamentals

Definition: A set of step-by-step instructions to solve a specific problem.

Why Algorithm Choice Matters:

  • Efficiency: Speed of execution
  • Space: Memory usage
  • Cost: Operational expenses
  • User Experience: App performance
  • Scalability: Growth capability

Search Methods Comparison

Method Time Sorted?
Linear Search O(n) No
Binary Search O(log n) YES ✓

For 1,000,000 elements:

  • Linear: 1,000,000 operations (17 min)
  • Binary: 20 operations (20 ms)

Big O Notation Hierarchy

O(1) < O(log n) < O(n) < O(n log n) < O(n²) < O(n³) ↑ ↑ ↑ ↑ ↑ ↑ Best Excellent Good Decent Acceptable Bad

Common Complexities:

  • O(1): Array access, math operations
  • O(log n): Binary search
  • O(n): Single loop, linear search
  • O(n²): Nested loops, bubble sort

Logarithms Quick Guide

Formula: log₂(x) = y ⟺ 2^y = x

Common Values:
log₂(8) = 3 (because 2³ = 8)
log₂(16) = 4 (because 2⁴ = 16)
log₂(1024) = 10 (because 2¹⁰ = 1024)

Why base 2? Computer science is binary!

⏱️ Time vs Space Complexity

Time Complexity:

How long does it take? Measures number of operations.

Space Complexity:

How much memory does it use? Measures memory allocation.

Time-Space Tradeoff: Optimizing one often increases the other. Choose based on your constraints!

Big O Calculation Rules

1. Drop Constants

O(2n) → O(n) O(500) → O(1)

2. Drop Lower-Order Terms

O(n² + n) → O(n²) O(n log n + n) → O(n log n)

3. Common Patterns

  • Single loop → O(n)
  • Nested loops → O(n²)
  • Halving each step → O(log n)

Binary Search Pseudocode

FUNCTION BinarySearch(array, target): low = 0 high = length(array) - 1 WHILE low <= high: mid = (low + high) / 2 IF array[mid] == target THEN RETURN mid ELSE IF array[mid] < target THEN low = mid + 1 ELSE high = mid - 1 END IF END WHILE RETURN -1 END FUNCTION

Requirement: Array MUST be sorted!

Key Takeaways

  • Binary search is MUCH faster than linear (O(log n) vs O(n))
  • Big O focuses on growth rate, not exact time
  • Binary search requires sorted data
  • For 1M elements: Linear = 17 min, Binary = 20 ms
  • Time-space tradeoffs exist in all algorithms
  • Drop constants and lower-order terms in Big O
