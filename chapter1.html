SAMENVATTING

Basis van algoritmen

Definitie: Een reeks stapsgewijze instructies om een specifiek probleem op te lossen.

Waarom de keuze van een algoritme belangrijk is:

  • Efficiëntie: Uitvoeringssnelheid
  • Ruimte: Geheugengebruik
  • Kost: Operationele kosten
  • Gebruikerservaring: App-prestaties
  • Schaalbaarheid: Groeicapaciteit

Vergelijking van zoekmethoden

Methode Tijd Gesorteerd?
Lineair zoeken O(n) Nee
Binair zoeken O(log n) JA ✓

Voor 1.000.000 elementen:

  • Lineair: 1.000.000 bewerkingen (17 min)
  • Binair: 20 bewerkingen (20 ms)

Hiërarchie van Big O-notatie

O(1) < O(log n) < O(n) < O(n log n) < O(n²) < O(n³) ↑ ↑ ↑ ↑ ↑ ↑ Best Uitstekend Goed Redelijk Aanvaardbaar Slecht

Veelvoorkomende complexiteiten:

  • O(1): Array-toegang, wiskundige bewerkingen
  • O(log n): Binair zoeken
  • O(n): Eén lus, lineair zoeken
  • O(n²): Geneste lussen, bubble sort

Korte gids voor logaritmen

Formule: log₂(x) = y ⟺ 2^y = x

Veelvoorkomende waarden:
log₂(8) = 3 (omdat 2³ = 8)
log₂(16) = 4 (omdat 2⁴ = 16)
log₂(1024) = 10 (omdat 2¹⁰ = 1024)

Waarom basis 2? Informatica is binair!

⏱️ Tijd- versus ruimtecomplexiteit

Tijdcomplexiteit:

Hoe lang duurt het? Meet het aantal bewerkingen.

Ruimtecomplexiteit:

Hoeveel geheugen gebruikt het? Meet de geheugenallocatie.

Afweging tussen tijd en ruimte: Het optimaliseren van het ene verhoogt vaak het andere. Kies op basis van je beperkingen!

Rekenregels voor Big O

1. Laat constanten weg

O(2n) → O(n) O(500) → O(1)

2. Laat termen van lagere orde weg

O(n² + n) → O(n²) O(n log n + n) → O(n log n)

3. Veelvoorkomende patronen

  • Eén lus → O(n)
  • Geneste lussen → O(n²)
  • Elke stap halveren → O(log n)

Pseudocode voor binair zoeken

FUNCTION BinarySearch(array, target): low = 0 high = length(array) - 1 WHILE low <= high: mid = (low + high) / 2 IF array[mid] == target THEN RETURN mid ELSE IF array[mid] < target THEN low = mid + 1 ELSE high = mid - 1 END IF END WHILE RETURN -1 END FUNCTION

Vereiste: De array MOET gesorteerd zijn!

Belangrijkste inzichten

  • Binair zoeken is VEEL sneller dan lineair zoeken (O(log n) vs O(n))
  • Big O focust op groeisnelheid, niet op exacte tijd
  • Binair zoeken vereist gesorteerde data
  • Voor 1M elementen: lineair = 17 min, binair = 20 ms
  • Afwegingen tussen tijd en ruimte bestaan in alle algoritmen
  • Laat constanten en termen van lagere orde weg in Big O
