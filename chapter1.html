Samenvatting

Een algoritme is een concreet stappenplan om een probleem op te lossen. In Programming Advanced is vooral belangrijk dat je niet enkel weet wat een algoritme is, maar dat je kunt uitleggen waarom de ene aanpak beter past dan de andere.

Searching betekent dat je een waarde zoekt in een verzameling data. Lineair zoeken werkt altijd, maar controleert in het slechtste geval elk element. Binary search is veel sneller, maar mag alleen gebruikt worden wanneer de data gesorteerd is.

Big O beschrijft hoe de tijd of het geheugengebruik groeit wanneer de input groter wordt. Je laat constanten en lagere termen weg, omdat Big O draait om groeigedrag.

Wat moet je begrijpen?

  • Lineair zoeken: O(n), geen gesorteerde input nodig.
  • Binary search: O(log n), vereist gesorteerde input.
  • Time complexity: aantal bewerkingen in functie van inputgrootte.
  • Space complexity: extra geheugen in functie van inputgrootte.
  • Volgorde: O(1) < O(log n) < O(n) < O(n log n) < O(n²) < O(2^n).

Big O & complexiteit

  • Array index access: O(1)
  • Eén lus over n elementen: O(n)
  • Twee geneste lussen: O(n²)
  • Telkens halveren: O(log n)
  • Sorteren met efficiënte sort: vaak O(n log n)

C# voorbeeld

static int BinarySearch(int[] a, int target)
{
    int low = 0;
    int high = a.Length - 1;

    while (low <= high)
    {
        int mid = low + (high - low) / 2;

        if (a[mid] == target) return mid;
        if (a[mid] < target) low = mid + 1;
        else high = mid - 1;
    }

    return -1;
}

Typische valkuilen

  • Binary search gebruiken op ongesorteerde data.
  • O(2n + 5) niet vereenvoudigen naar O(n).
  • Exacte runtime verwarren met Big O.
  • Best-case en worst-case door elkaar halen.
