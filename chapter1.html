SAMENVATTING SECTIE

Algoritmes Fundamentals

Definitie: Een reeks stap-voor-stap instructies om een specifiek probleem op te lossen.

Waarom de keuze van algoritme belangrijk is:

  • Efficiency: Uitvoeringssnelheid
  • Geheugen: Geheugengebruik
  • Kost: Operationele kosten
  • User Experience: App-performance
  • Scalability: Schaalbaarheid/groeimogelijkheid

Vergelijking van zoekmethodes

Methode Tijd Gesorteerd?
Lineair zoeken O(n) Nee
Binary search O(log n) JA ✓

Voor 1.000.000 elementen:

  • Lineair: 1.000.000 bewerkingen (17 min)
  • Binary: 20 bewerkingen (20 ms)

Hiërarchie van Big O-notatie

O(1) < O(log n) < O(n) < O(n log n) < O(n²) < O(n³) ↑ ↑ ↑ ↑ ↑ ↑ Best Uitstekend Goed Redelijk Aanvaardbaar Slecht

Veelvoorkomende complexiteiten:

  • O(1): Array-toegang, wiskundige bewerkingen
  • O(log n): Binary search
  • O(n): Eén lus, lineair zoeken
  • O(n²): Geneste lussen, bubble sort

Snelle gids voor logaritmen

Formule: log₂(x) = y ⟺ 2^y = x

Veelvoorkomende waarden:
log₂(8) = 3 (omdat 2³ = 8)
log₂(16) = 4 (omdat 2⁴ = 16)
log₂(1024) = 10 (omdat 2¹⁰ = 1024)

Waarom basis 2? In computerwetenschappen is alles binair!

⏱️ Tijdscomplexiteit vs geheugencomplexiteit

Tijdscomplexiteit:

Hoe lang duurt het? Meet het aantal bewerkingen.

Geheugencomplexiteit:

Hoeveel geheugen gebruikt het? Meet de geheugenallocatie.

Time-space tradeoff: Het ene optimaliseren vergroot vaak het andere. Kies op basis van je constraints!

Regels om Big O te berekenen

1. Laat constanten weg

O(2n) → O(n) O(500) → O(1)

2. Laat lagere termen weg

O(n² + n) → O(n²) O(n log n + n) → O(n log n)

3. Veelvoorkomende patronen

  • Eén lus → O(n)
  • Geneste lussen → O(n²)
  • Elke stap halveren → O(log n)

Binary search pseudocode

Functie BinarySearch(array, doel): low = 0 high = length(array) - 1 WHILE low <= high: mid = (low + high) / 2 IF array[mid] == doel THEN RETURN mid ELSE IF array[mid] < doel THEN low = mid + 1 ELSE high = mid - 1 END IF END WHILE RETURN -1 END Functie

Vereiste: De array MOET gesorteerd zijn!

Belangrijkste inzichten

  • Binary search is VEEL sneller dan lineair zoeken (O(log n) vs O(n))
  • Big O focust op groeisnelheid, niet op exacte tijd
  • Binary search vereist gesorteerde data
  • Voor 1 miljoen elementen: lineair = 17 min, binary = 20 ms
  • Time-space tradeoffs bestaan in alle algoritmes
  • Laat constanten en lagere termen weg in Big O
