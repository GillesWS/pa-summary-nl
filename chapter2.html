SAMENVATTING

Waarom sorteren belangrijk is

Definitie: Items ordenen in een specifieke volgorde (oplopend/aflopend)

Belangrijkste voordelen:

  • Sneller zoeken: Maakt binary search O(log n) mogelijk
  • Data-analyse: Patronen worden zichtbaar
  • Voorwaarden: Veel algoritmes hebben gesorteerde input nodig
  • Presentatie: Mensen verkiezen geordende data

Bubble sort

Strategie: Wissel herhaaldelijk aangrenzende elementen als ze fout staan

FOR i = 0 TO n-2: FOR j = 0 TO n-2-i: IF A[j] > A[j+1]: Swap A[j], A[j+1]

Complexiteit:

  • Tijd: O(n²) worst-case, O(n) best-case met flag
  • Geheugen: O(1)
  • Stabiel: Ja
Optimalisatie: Gebruik een flag - als er geen swaps zijn in een pass, stop vroeg!

Selection sort

Strategie: Zoek het minimum en plaats het vooraan

FOR i = 0 TO n-2: minIdx = i FOR j = i+1 TO n-1: IF A[j] < A[minIdx]: minIdx = j Swap A[i], A[minIdx]

Complexiteit:

  • Tijd: Altijd O(n²) - geen betere best-case!
  • Geheugen: O(1)
  • Stabiel: Nee
Zwakte: Doet altijd maximaal werk. Vermijden!

Insertion sort

Strategie: Bouw de gesorteerde array één element per keer op. De "menselijke manier"!

FOR i = 1 TO n-1: key = A[i] j = i-1 WHILE j >= 0 AND A[j] > key: A[j+1] = A[j] j-- A[j+1] = key

Complexiteit:

  • Tijd: O(n²) worst-case, O(n) best-case
  • Geheugen: O(1)
  • Stabiel: Ja
Meest intelligent! Past zich aan gedeeltelijk gesorteerde data aan

Pruning

Definitie: Onnodig werk wegknippen

Technieken:

  • Vroeg stoppen: Stop wanneer het doel bereikt is
  • Flags: Hou de status bij van iets klaar is
  • Bounds checking: Sla onmogelijke bereiken over
Voorbeeld: Flag in Bubble Sort → O(n²) naar O(n) voor gesorteerde data!

Belangrijkste inzichten

  • Alle drie basis-sorteeralgoritmes zijn O(n²)
  • Insertion sort is het slimst - past zich aan de data aan
  • Selection sort is altijd traag - vermijden!
  • Bubble sort kan geoptimaliseerd worden met flags
  • Alle drie sorteren in-place: O(1) geheugen
  • Pruning = slimmer werken, niet harder
  • Stabiliteit behoudt de volgorde van gelijke elementen

Tijdscomplexiteit van Bubble Sort?▼
O(n²) worst/average-case, O(n) beste case met flag optimalisatie
Hoe werkt Bubble Sort?▼
Wisselt herhaaldelijk aangrenzende elementen als ze fout in de juiste volgorde, "bubbling" de grootste element naar het einde
Complexiteit van Selection Sort?▼
Altijd O(n²) - geen betere best-case! Doet altijd maximaal werkt
Hoe werkt Selection Sort?▼
Finds de minimum element en places it at de beginning, repeats voor remaining array
Insertion Sort beste case?▼
O(n) voor gesorteerd of nearly gesorteerde data
Hoe werkt Insertion Sort?▼
Builds gesorteerd array one element at een tijd door inserting elke element into its juiste plaats - de "menselijke manier"!
Wat is pruning?▼
Onnodig werk wegknippen naar make algoritmes smarter en more efficiënt
Wat is een flag?▼
Boolean variable used naar track state/status en control lus termination
Wat is stable sorteren?▼
Behoudt de relatief volgorde van gelijke elementen van de oorspronkelijke array
Most intelligent basic sort?▼
Insertion Sort - adapts naar partially gesorteerde data met natural early stopping
