02. Sorting Algorithms

Bubble sort, selection sort, insertion sort en pruning

Samenvatting

Sorting betekent dat je data ordent, bijvoorbeeld van klein naar groot. Dat maakt zoeken, analyseren en verdere algoritmes vaak eenvoudiger.

Bubble sort vergelijkt buren en wisselt ze om. Selection sort zoekt telkens het minimum. Insertion sort bouwt een gesorteerd deel op en voegt elk nieuw element op de juiste plaats in.

Pruning betekent dat je onnodig werk stopt. Bij bubble sort kan een flag detecteren dat er geen swaps meer gebeuren, waardoor een gesorteerde lijst sneller klaar is.

Wat moet je begrijpen?

  • Bubble sort: adjacent swaps, O(n²) worst-case, O(n) best-case met flag.
  • Selection sort: zoekt telkens minimum, altijd O(n²).
  • Insertion sort: goed bij bijna gesorteerde data, O(n) best-case.
  • Stable sorting: gelijke elementen behouden hun onderlinge volgorde.
  • In-place sorting: gebruikt weinig extra geheugen.

Big O & complexiteit

  • Bubble sort: O(n²) worst/average, O(n) best met flag.
  • Selection sort: altijd O(n²).
  • Insertion sort: O(n²) worst, O(n) best.
  • Space voor deze drie simpele sorts: O(1).

C# voorbeeld

static void InsertionSort(int[] a)
{
    for (int i = 1; i < a.Length; i++)
    {
        int key = a[i];
        int j = i - 1;

        while (j >= 0 && a[j] > key)
        {
            a[j + 1] = a[j];
            j--;
        }

        a[j + 1] = key;
    }
}

Typische valkuilen

  • Selection sort als slim of adaptief beschouwen; dat is het niet.
  • Vergeten dat insertion sort zeer goed kan zijn bij bijna gesorteerde data.
  • De flag bij bubble sort verkeerd initialiseren.
  • Stable en in-place verwarren.
