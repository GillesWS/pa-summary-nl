SAMENVATTING

Waarom sorteren belangrijk is

Definitie: Items ordenen in een specifieke volgorde (oplopend/aflopend)

Belangrijkste voordelen:

  • Sneller zoeken: Maakt binair zoeken O(log n) mogelijk
  • Data-analyse: Patronen worden zichtbaar
  • Vereisten: Veel algoritmen hebben gesorteerde invoer nodig
  • Presentatie: Mensen verkiezen georganiseerde data

Bubble sort

Strategie: Wissel aangrenzende elementen herhaaldelijk om als ze niet in de juiste volgorde staan

FOR i = 0 TO n-2: FOR j = 0 TO n-2-i: IF A[j] > A[j+1]: Swap A[j], A[j+1]

Complexiteit:

  • Tijd: O(n²) slechtste geval, O(n) beste geval met flag
  • Ruimte: O(1)
  • Stable: Ja
Optimalisatie: Gebruik een flag: als er in een ronde geen swaps zijn, stop dan vroegtijdig!

Selection sort

Strategie: Zoek het minimum en plaats het vooraan

FOR i = 0 TO n-2: minIdx = i FOR j = i+1 TO n-1: IF A[j] < A[minIdx]: minIdx = j Swap A[i], A[minIdx]

Complexiteit:

  • Tijd: Altijd O(n²) - geen beste geval!
  • Ruimte: O(1)
  • Stable: Nee
Zwakte: Doet altijd maximaal werk. Vermijd dit!

Insertion sort

Strategie: Bouw de gesorteerde array één element per keer op. De "menselijke manier"!

FOR i = 1 TO n-1: key = A[i] j = i-1 WHILE j >= 0 AND A[j] > key: A[j+1] = A[j] j-- A[j+1] = key

Complexiteit:

  • Tijd: O(n²) slechtste geval, O(n) beste geval
  • Ruimte: O(1)
  • Stable: Ja
Meest intelligent! Past zich aan aan gedeeltelijk gesorteerde data

Pruning

Definitie: Onnodig werk wegsnijden

Technieken:

  • Vroegtijdig stoppen: Stop wanneer het doel bereikt is
  • Flags: Houd de voltooiingsstatus bij
  • Bounds checking: Sla onmogelijke bereiken over
Voorbeeld: Flag in bubble sort → O(n²) naar O(n) voor gesorteerde data!

Belangrijkste inzichten

  • Alle drie de basissorteeralgoritmen zijn O(n²)
  • Insertion sort is het meest intelligent: past zich aan de data aan
  • Selection sort is altijd traag: vermijd dit!
  • Bubble sort kan met flags geoptimaliseerd worden
  • Alle sorteren in-place: O(1) ruimte
  • Pruning = slimmer werken, niet harder
  • Stabiliteit behoudt de volgorde van gelijke elementen

Wat is de tijdcomplexiteit van bubble sort?▼

Hoe werkt bubble sort?▼
Wisselt aangrenzende elementen herhaaldelijk om als ze niet juist staan, waardoor het grootste element naar het einde "bubbelt"
Wat is de complexiteit van selection sort?▼
Altijd O(n²) - geen beste geval! Doet altijd maximaal werk
Hoe werkt selection sort?▼
Zoekt het kleinste element, plaatst het vooraan en herhaalt dit voor de rest van de array
Wat is het beste geval van insertion sort?▼
O(n) voor gesorteerde of bijna gesorteerde data
Hoe werkt insertion sort?▼
Bouwt de gesorteerde array één element per keer op door elk element op de juiste plaats in te voegen - de "menselijke manier"!
Wat is pruning?▼
Onnodig werk wegsnijden om algoritmen slimmer en efficiënter te maken
Wat is een flag?▼
Booleaanse variabele die gebruikt wordt om toestand/status bij te houden en lussen te stoppen
Wat is stabiel sorteren?▼
Behoudt de relatieve volgorde van gelijke elementen uit de oorspronkelijke array
Wat is het meest intelligente basissorteeralgoritme?▼
Insertion sort - past zich aan gedeeltelijk gesorteerde data aan met natuurlijke vroegtijdige stop
