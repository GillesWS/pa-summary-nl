Why Sorting Matters

Definition: Organizing items in specific order (ascending/descending)

Key Benefits:

  • Faster Searching: Enables binary search O(log n)
  • Data Analysis: Patterns become visible
  • Prerequisites: Many algorithms need sorted input
  • Presentation: Humans prefer organized data

Bubble Sort

Strategy: Repeatedly swap adjacent elements if out of order

FOR i = 0 TO n-2: FOR j = 0 TO n-2-i: IF A[j] > A[j+1]: Swap A[j], A[j+1]

Complexity:

  • Time: O(n²) worst, O(n) best with flag
  • Space: O(1)
  • Stable: Yes
Optimization: Use flag - if no swaps in pass, stop early!

Selection Sort

Strategy: Find minimum, place at beginning

FOR i = 0 TO n-2: minIdx = i FOR j = i+1 TO n-1: IF A[j] < A[minIdx]: minIdx = j Swap A[i], A[minIdx]

Complexity:

  • Time: Always O(n²) - no best case!
  • Space: O(1)
  • Stable: No
Weakness: Always does maximum work. Avoid!

Insertion Sort

Strategy: Build sorted array one element at a time. The "human way"!

FOR i = 1 TO n-1: key = A[i] j = i-1 WHILE j >= 0 AND A[j] > key: A[j+1] = A[j] j-- A[j+1] = key

Complexity:

  • Time: O(n²) worst, O(n) best
  • Space: O(1)
  • Stable: Yes
Most Intelligent! Adapts to partially sorted data

Pruning

Definition: Cutting away unnecessary work

Techniques:

  • Early Termination: Stop when goal reached
  • Flags: Track completion status
  • Bounds Checking: Skip impossible ranges
Example: Flag in Bubble Sort → O(n²) to O(n) for sorted data!

Key Takeaways

  • All three basic sorts are O(n²)
  • Insertion Sort most intelligent - adapts to data
  • Selection Sort always slow - avoid!
  • Bubble Sort can be optimized with flags
  • All sort in-place: O(1) space
  • Pruning = work smarter not harder
  • Stability preserves order of equal elements
