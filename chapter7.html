07. Dynamic Programming

Memoization, tabulation en overlapping subproblems

Samenvatting

Dynamic Programming optimaliseert oplossingen door resultaten van subproblemen op te slaan.

DP gebruik je wanneer dezelfde subproblemen vaker terugkomen en wanneer een optimale oplossing kan worden opgebouwd uit optimale deeloplossingen.

Memoization werkt top-down met recursie en cache. Tabulation werkt bottom-up met een tabel.

Wat moet je begrijpen?

  • Overlapping subproblems: dezelfde berekening komt terug.
  • Optimal substructure: optimale oplossing bestaat uit optimale deeloplossingen.
  • Memoization: recursie + cache.
  • Tabulation: tabel van klein naar groot invullen.

Big O & complexiteit

  • Naive Fibonacci: O(2^n).
  • Fibonacci memoization: O(n).
  • Fibonacci tabulation: O(n).
  • Knapsack DP: O(n * W).

C# voorbeeld

static int FibMemo(int n, int[] memo)
{
    if (n <= 1) return n;
    if (memo[n] != -1) return memo[n];

    memo[n] = FibMemo(n - 1, memo) + FibMemo(n - 2, memo);
    return memo[n];
}

Typische valkuilen

  • Cache verkeerd initialiseren.
  • DP gebruiken zonder overlappende subproblemen.
  • Off-by-one fouten in tabellen.
  • Vergeten dat DP vaak extra geheugen kost.
