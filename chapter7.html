Wat is dynamisch programmeren?

Definitie: Optimalisatietechniek voor recursie door tussentijdse resultaten op te slaan.

Hoofdidee:

  • Sla resultaten van deelproblemen op
  • Hergebruik opgeslagen resultaten in plaats van opnieuw te berekenen
  • Zet exponentiële tijd om naar polynomiale tijd
Belangrijk inzicht: Los hetzelfde probleem niet twee keer op!

Top-down: Memoization

Aanpak: Start bovenaan en sla resultaten op terwijl je naar beneden gaat

Hoe het werkt:

  • Gebruik een lookup-tabel (array/dictionary)
  • Controleer of het resultaat al berekend is
  • Zo ja, geef het opgeslagen resultaat terug
  • Zo nee, bereken en sla op
int[] memo; int FibMemo(int n) { if (n <= 1) return n; if (memo[n] > 0) return memo[n]; memo[n] = FibMemo(n-1) + FibMemo(n-2); return memo[n]; }

Complexiteit: O(n) tijd, O(n) ruimte

Bottom-up: tabulation

Aanpak: Start onderaan en bouw op naar de oplossing

Hoe het werkt:

  • Vul de tabel van klein naar groot
  • Geen recursie nodig
  • Iteratieve aanpak
int FibTab(int n) { int[] tab = new int[n + 2]; tab[0] = 0; tab[1] = 1; for (int i = 2; i <= n; i++) { tab[i] = tab[i-1] + tab[i-2]; } return tab[n]; }

Complexiteit: O(n) tijd, O(n) ruimte

🆚 Memoization versus tabulation

Memoization (top-down):

  • + Vul de tabel wanneer nodig
  • + Natuurlijke recursieve aanpak
  • + Bereken alleen de nodige deelproblemen
  • - Recursie-overhead

Tabulation (bottom-up):

  • + Geen recursie-overhead
  • + Meestal sneller
  • - Moet de volgorde kennen
  • - Berekent alle deelproblemen

Fibonacci-vergelijking

Aanpak Tijd Ruimte
Naïeve recursie O(2^n) O(n)
Memoization O(n) O(n)
Tabulation O(n) O(n)
Iteratief O(n) O(1)

Wanneer gebruik je DP

Twee vereisten:

  • Overlappende deelproblemen: Dezelfde problemen worden meerdere keren opgelost
  • Optimale substructuur: Een optimale oplossing bevat optimale oplossingen van deelproblemen

Klassieke DP-problemen:

  • Fibonacci-reeks
  • Knapsackprobleem
  • Langste gemeenschappelijke deelreeks
  • Rod cutting
  • Muntwissel

️ Rod cutting-probleem

Probleem: Snijd een staaf van lengte n om de winst te maximaliseren

Given:

  • Staaflengte n
  • Prijs voor elke lengte (1 tot n)

Doel:

Vind de optimale manier om de staaf te snijden voor maximale opbrengst

Example: n=4 Prices: [1, 5, 8, 9] Options: - No cuts: 9 - Cut 1+3: 1+8=9 - Cut 2+2: 5+5=10 ✓ (optimaal) - Others... Formula: r(n) = max(price[i] + r(n-i)) for i=1 to n

Belangrijkste inzichten

  • DP = optimalisatie voor recursie
  • Sla tussentijdse resultaten op om herberekening te vermijden
  • Memoization: top-down met recursie
  • Tabulation: bottom-up iteratief
  • Vereist overlappende deelproblemen
  • Vereist optimale substructuur
  • Zet O(2^n) om naar O(n)
  • Ruil ruimte in voor tijd
