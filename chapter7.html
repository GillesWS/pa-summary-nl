Wat is Dynamic Programming?

Definitie: Optimalisatie technique voor recursie door storing intermediate results.

Main Idea:

  • Opslaan results van subproblemen
  • Reuse stored results instead van recalculating
  • Transform exponentieel naar polynomial tijd
Key Insight: Don't oplossen dezelfde probleem twice!

Top-Down: Memoization

Approach: Start van top, opslaan results als you go down

How it Werkt:

  • Gebruik lookup tabel (array/dictionary)
  • Check if result already computed
  • If yes, return stored result
  • If no, compute en opslaan
int[] memo; int FibMemo(int n) { if (n <= 1) return n; if (memo[n] > 0) return memo[n]; memo[n] = FibMemo(n-1) + FibMemo(n-2); return memo[n]; }

Complexiteit: O(n) tijd, O(n) geheugen

Bottom-Up: Tabulation

Approach: Start van bottom, bouw up naar oplossing

How it Werkt:

  • Fill tabel van smallest naar largest
  • Nee recursie needed
  • Iterative approach
int FibTab(int n) { int[] tab = new int[n + 2]; tab[0] = 0; tab[1] = 1; voor (int i = 2; i <= n; i++) { tab[i] = tab[i-1] + tab[i-2]; } return tab[n]; }

Complexiteit: O(n) tijd, O(n) geheugen

🆚 Memoization vs Tabulation

Memoization (Top-Down):

  • + Fill tabel on demand
  • + Natural recursief approach
  • + Only compute needed subproblemen
  • - Recursie overhead

Tabulation (Bottom-Up):

  • + Nee recursie overhead
  • + Usually sneller
  • - Must know volgorde
  • - Computes alle subproblemen

Fibonacci Vergelijking

Approach Tijd Geheugen
Naive Recursie O(2^n) O(n)
Memoization O(n) O(n)
Tabulation O(n) O(n)
Iterative O(n) O(1)

Wanneer naar Gebruik DP

Two Requirements:

  • Overlapping Subproblemen: Same problemen solved multiple times
  • Optimal Substructure: Een optimale oplossing bevat optimale oplossingen voor subproblemen

Classic DP Problemen:

  • Fibonacci sequence
  • Knapsack probleem
  • Longest veelvoorkomend subsequence
  • Rod cutting
  • Coin change

️ Rod Cutting Probleem

Probleem: Cut rod van length n naar maximize profit

Given:

  • Rod length n
  • Price voor elke length (1 naar n)

Goal:

Find optimal way naar cut rod voor maximaalm revenue

Voorbeeld: n=4 Prices: [1, 5, 8, 9] Options: - Nee cuts: 9 - Cut 1+3: 1+8=9 - Cut 2+2: 5+5=10 ✓ (optimal) - Others... Formule: r(n) = max(price[i] + r(n-i)) voor i=1 naar n

Belangrijkste inzichten

  • DP = optimalisatie voor recursie
  • Opslaan intermediate results naar avoid recomputation
  • Memoization: top-down met recursie
  • Tabulation: bottom-up iterative
  • Vereist overlapping subproblemen
  • Vereist optimale substructuur
  • Transforms O(2^n) naar O(n)
  • Trade geheugen voor tijd
