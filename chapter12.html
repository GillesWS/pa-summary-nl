12. Hash Tables

Hash functions, collisions en snelle lookups

Home 01 02 03 04 05 06 07 08 09 10 11 12 13

Samenvatting

Een hash table combineert een hash function met een array om key-value data snel terug te vinden.

Gemiddeld zijn search, insert en delete O(1), maar collisions kunnen de performance slechter maken.

Collision handling kan bijvoorbeeld via open addressing of separate chaining.

Wat moet je begrijpen?

  • Hash function: zet key om naar index.
  • Collision: twee keys krijgen dezelfde index.
  • Load factor: hoeveel de tabel gevuld is.
  • Separate chaining: per slot een lijst van items.
  • Open addressing: vrije plaats zoeken in de array.

Big O & complexiteit

  • Gemiddelde lookup: O(1).
  • Worst-case lookup bij veel collisions: O(n).
  • Resize/rehash kan tijdelijk duur zijn.
  • Dictionary in C# is meestal hash table-gebaseerd.

C# voorbeeld

var counts = new Dictionary<string, int>();

foreach (string word in words)
{
    if (!counts.ContainsKey(word))
        counts[word] = 0;

    counts[word]++;
}

Typische valkuilen

  • Denken dat collisions niet bestaan.
  • Dictionary als gesorteerde structuur beschouwen.
  • Slechte hash function of te kleine tabel gebruiken.
  • Gemiddelde O(1) verwarren met gegarandeerde worst-case O(1).
