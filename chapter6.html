Wat is backtracking?

Definitie: Systematische trial-and-error-aanpak om alle mogelijke oplossingen te verkennen.

Kernidee:

  • Probeer een keuze → als die werkt, ga dieper verder
  • Als die faalt → backtrack en probeer een andere keuze
  • Verken alle mogelijkheden tot een oplossing gevonden is
Belangrijk inzicht: Systematische verkenning met de mogelijkheid om keuzes ongedaan te maken!

N-Queens-probleem

Doel: Plaats N koninginnen op een N×N-schaakbord zodat geen twee koninginnen elkaar bedreigen.

Beperkingen:

  • Geen twee koninginnen in dezelfde rij
  • Geen twee koninginnen in dezelfde kolom
  • Geen twee koninginnen op dezelfde diagonaal

Algoritme:

1. Start in de meest linkse kolom 2. If all queens placed → return true 3. For each row in current column: - If queen can be placed safely → mark it - Recurseer om de overige koninginnen te plaatsen - If leads to solution → return true - Else → unmark (backtrack) and try next row

Subsetprobleem

Doel: Genereer alle subsets van een gegeven set.

Voorbeeld:

Input: [1, 2, 3]
Output: {}, {1}, {2}, {3}, {1,2}, {1,3}, {2,3}, {1,2,3}

Backtracking-aanpak:

1. Start met een lege subset 2. For each element: - Include element (recurse) - Exclude element (backtrack)

Aantal subsets:

Voor n elementen: 2^n subsets (elk element: opnemen of uitsluiten)

Binaire aanpak (subsets)

Alternatief: Gebruik binaire representatie

For [1,2,3]: 000 → {} 001 → {3} 010 → {2} 011 → {2,3} 100 → {1} 101 → {1,3} 110 → {1,2} 111 → {1,2,3}

Elke bit betekent opnemen (1) of uitsluiten (0)

Backtracking-patroon

FUNCTION Backtrack(state): IF is_solution(state) THEN add_to_results(state) RETURN END IF FOR each choice in choices: IF is_valid(choice) THEN make_choice(choice) Backtrack(new_state) undo_choice(choice) // Backtrack! END IF END FOR END FUNCTION

Belangrijke onderdelen:

  • Keuze: Welke opties hebben we?
  • Beperking: Wat maakt een keuze geldig?
  • Doel: Wanneer hebben we een oplossing gevonden?

Wanneer gebruik je backtracking?

Perfect voor:

  • Constraintproblemen: Sudoku, N-Queens
  • Permutaties/combinaties: Genereer alle mogelijkheden
  • Subsetgeneratie: Alle combinaties van elementen
  • Pad zoeken: Doolhoven, puzzels
  • Spellen oplossen: Boter-kaas-en-eieren, schaakzetten

Kenmerken:

  • Geen direct algoritme naar de oplossing
  • Moet meerdere mogelijkheden verkennen
  • Moet keuzes proberen en mogelijk ongedaan maken

Optimalisatie: pruning

Idee: Knip takken weg die niet tot een geldige oplossing kunnen leiden

Voordelen:

  • Vermijd vroegtijdig het verkennen van ongeldige paden
  • Verminder de zoekruimte drastisch
  • Maak backtracking praktisch

Voorbeeld - N-Queens:

Controleer VOOR de recursie of de plaatsing van een koningin veilig is (plaats niet als ze bestaande koninginnen bedreigt)

Belangrijkste inzichten

  • Backtracking = systematische trial-and-error
  • Gebruikt recursie om mogelijkheden te verkennen
  • Maak keuze → recursie → maak ongedaan als het faalt
  • Essentieel voor constraint satisfaction problems
  • Pruning verbetert efficiëntie drastisch
  • N-Queens: klassiek backtrackingvoorbeeld
  • Subsets: 2^n mogelijkheden (elk opnemen/uitsluiten)
  • Patroon: kies → verken → backtrack
