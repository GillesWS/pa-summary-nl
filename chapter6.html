Wat is Backtracking?

Definitie: Systematic trial-and-error approach naar explore alle mogelijk oplossingen.

Core Idea:

  • Try een keuze → if werkt, continue deeper
  • If faalt → backtrack en try different keuze
  • Explore alle possibilities until oplossing found
Key Insight: Systematic exploration met ability naar undo keuzes!

N-Queens Probleem

Goal: Place N queens on N×N chessboard so no two queens threaten elke other.

Constraints:

  • Nee two queens in dezelfde row
  • Nee two queens in dezelfde column
  • Nee two queens on dezelfde diagonal

Algoritme:

1. Start in leftmost column 2. If alle queens placed → return waar 3. Voor elke row in huidige column: - If queen can be placed safely → mark it - Recurse naar place remaining queens - If leads naar oplossing → return waar - Else → unmark (backtrack) en try next row

Subset Probleem

Goal: Generate alle subsets van een given set.

Voorbeeld:

Input: [1, 2, 3]
Output: {}, {1}, {2}, {3}, {1,2}, {1,3}, {2,3}, {1,2,3}

Backtracking Approach:

1. Start met empty subset 2. Voor elke element: - Include element (recurse) - Exclude element (backtrack)

Number van Subsets:

Voor n elementen: 2^n subsets (elke element: include of exclude)

Binary Approach (Subsets)

Alternative: Gebruik binary representation

Voor [1,2,3]: 000 → {} 001 → {3} 010 → {2} 011 → {2,3} 100 → {1} 101 → {1,3} 110 → {1,2} 111 → {1,2,3}

Elke bit stelt voor include (1) of exclude (0)

Backtracking Patroon

Functie Backtrack(state): IF is_solution(state) THEN add_to_results(state) RETURN END IF Voor elke keuze in keuzes: IF is_valid(keuze) THEN make_choice(keuze) Backtrack(new_state) undo_choice(keuze) // Backtrack! END IF END Voor END Functie

Key Components:

  • Keuze: What options do we have?
  • Constraint: What makes keuze valid?
  • Goal: Wanneer have we found oplossing?

Wanneer naar Gebruik Backtracking

Perfect Voor:

  • Constraint problemen: Sudoku, N-Queens
  • Permutations/Combinations: Generate alle possibilities
  • Subset generation: Alle combinations van elementen
  • Pad finding: Mazes, puzzles
  • Game solving: Tic-tac-toe, chess moves

Characteristics:

  • Nee direct algoritme naar oplossing
  • Must explore multiple possibilities
  • Nodig hebben naar try en potentially undo keuzes

Optimalisatie: Pruning

Idea: Cut branches dat can't lead naar valid oplossing

Benefits:

  • Avoid exploring invalid paths early
  • Dramatically reduce zoeken geheugen
  • Make backtracking practical

Example - N-Queens:

Check if queen placement is safe BEFORE recursing (don't place if threatens existing queens)

Belangrijkste inzichten

  • Backtracking = systematic trial en error
  • Gebruikt recursie naar explore possibilities
  • Make keuze → recurse → undo if faalt
  • Essential voor constraint satisfaction problemen
  • Pruning improves efficiency dramatically
  • N-Queens: classic backtracking example
  • Subsets: 2^n possibilities (include/exclude elke)
  • Patroon: choose → explore → backtrack
