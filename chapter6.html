What is Backtracking?

Definition: Systematic trial-and-error approach to explore all possible solutions.

Core Idea:

  • Try a choice → if works, continue deeper
  • If fails → backtrack and try different choice
  • Explore all possibilities until solution found
Key Insight: Systematic exploration with ability to undo choices!

N-Queens Problem

Goal: Place N queens on N×N chessboard so no two queens threaten each other.

Constraints:

  • No two queens in same row
  • No two queens in same column
  • No two queens on same diagonal

Algorithm:

1. Start in leftmost column 2. If all queens placed → return true 3. For each row in current column: - If queen can be placed safely → mark it - Recurse to place remaining queens - If leads to solution → return true - Else → unmark (backtrack) and try next row

Subset Problem

Goal: Generate all subsets of a given set.

Example:

Input: [1, 2, 3]
Output: {}, {1}, {2}, {3}, {1,2}, {1,3}, {2,3}, {1,2,3}

Backtracking Approach:

1. Start with empty subset 2. For each element: - Include element (recurse) - Exclude element (backtrack)

Number of Subsets:

For n elements: 2^n subsets (each element: include or exclude)

Binary Approach (Subsets)

Alternative: Use binary representation

For [1,2,3]: 000 → {} 001 → {3} 010 → {2} 011 → {2,3} 100 → {1} 101 → {1,3} 110 → {1,2} 111 → {1,2,3}

Each bit represents include (1) or exclude (0)

Backtracking Pattern

FUNCTION Backtrack(state): IF is_solution(state) THEN add_to_results(state) RETURN END IF FOR each choice in choices: IF is_valid(choice) THEN make_choice(choice) Backtrack(new_state) undo_choice(choice) // Backtrack! END IF END FOR END FUNCTION

Key Components:

  • Choice: What options do we have?
  • Constraint: What makes choice valid?
  • Goal: When have we found solution?

When to Use Backtracking

Perfect For:

  • Constraint problems: Sudoku, N-Queens
  • Permutations/Combinations: Generate all possibilities
  • Subset generation: All combinations of elements
  • Path finding: Mazes, puzzles
  • Game solving: Tic-tac-toe, chess moves

Characteristics:

  • No direct algorithm to solution
  • Must explore multiple possibilities
  • Need to try and potentially undo choices

Optimization: Pruning

Idea: Cut branches that can't lead to valid solution

Benefits:

  • Avoid exploring invalid paths early
  • Dramatically reduce search space
  • Make backtracking practical

Example - N-Queens:

Check if queen placement is safe BEFORE recursing (don't place if threatens existing queens)

Key Takeaways

  • Backtracking = systematic trial and error
  • Uses recursion to explore possibilities
  • Make choice → recurse → undo if fails
  • Essential for constraint satisfaction problems
  • Pruning improves efficiency dramatically
  • N-Queens: classic backtracking example
  • Subsets: 2^n possibilities (include/exclude each)
  • Pattern: choose → explore → backtrack
