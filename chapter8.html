Samenvatting

Een graph bestaat uit nodes/vertices en edges. Graphs modelleren relaties, routes, netwerken en afhankelijkheden.

DFS zoekt diep en gebruikt recursie of een stack. BFS zoekt breed en gebruikt een queue.

Visited is essentieel om cycles en infinite loops te vermijden.

Wat moet je begrijpen?

  • Vertex/node: punt in de graph.
  • Edge: verbinding tussen nodes.
  • Adjacency list: per node een lijst van buren.
  • Adjacency matrix: 2D-array met verbindingen.
  • DFS: depth-first search.
  • BFS: breadth-first search.

Big O & complexiteit

  • BFS: O(V + E).
  • DFS: O(V + E).
  • Adjacency list space: O(V + E).
  • Adjacency matrix space: O(V²).

C# voorbeeld

static void BFS(List<int>[] graph, int start)
{
    bool[] visited = new bool[graph.Length];
    Queue<int> q = new Queue<int>();

    visited[start] = true;
    q.Enqueue(start);

    while (q.Count > 0)
    {
        int node = q.Dequeue();

        foreach (int next in graph[node])
        {
            if (!visited[next])
            {
                visited[next] = true;
                q.Enqueue(next);
            }
        }
    }
}

Typische valkuilen

  • Visited vergeten.
  • BFS met stack implementeren; dat wordt DFS.
  • DFS gebruiken wanneer shortest path in unweighted graph nodig is.
  • Adjacency matrix kiezen bij zeer sparse graph zonder reden.
