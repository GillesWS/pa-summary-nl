What zijn Graphs?

Definitie: Non-linear data structure met vertices (nodes) en edges (connections)

Components:

  • Vertices/Nodes: Points in graph
  • Edges: Connections between vertices

Types:

  • Directed: One-way edges
  • Undirected: Two-way edges
  • Weighted: Edges have kosten
  • Unweighted: Alle edges gelijke

Een lot van different graphs

Types van Graphs:

  • Undirected: Relationships zijn mutual (eg roads between cities)
  • Directed: Edges indicate een direction (like parents/children)
  • Disconnected: One of more nodes zijn not connected (like continents)
  • Acyclic: Nee cycles
  • Complete: Elke node is connected naar elke other node
  • Weighted: Edges between nodes have een gewicht

DFS - Depth First Zoeken

Strategie: Go deep, explore one pad completely before backtracking

Implementation:

  • Gebruik Stack (of recursie)
  • Track visited nodes
  • LIFO volgorde
DFS(node, visited): mark node als visited Voor elke neighbor van node: IF neighbor not visited: DFS(neighbor, visited)

Complexiteit: O(V + E) where V=vertices, E=edges

BFS - Breadth First Zoeken

Strategie: Go wide, explore alle neighbors before moving deeper

Implementation:

  • Gebruik Queue
  • Track visited nodes
  • FIFO volgorde
BFS(start): queue = [start] mark start als visited WHILE queue not empty: node = dequeue() Voor elke neighbor: IF not visited: mark visited enqueue(neighbor)

Complexiteit: O(V + E)

🆚 DFS vs BFS

DFS (Stack/Recursion):

  • + Less geheugen voor deep graphs
  • + Goed voor detecting cycles
  • - Doesn't guarantee kortste pad

BFS (Queue):

  • + Finds kortste pad (unweighted)
  • + Level-by-level exploration
  • - More geheugen voor wide graphs

Graph Representations

Adjacency Matrix:

  • 2D array [V][V]
  • Geheugen: O(V²)
  • Check edge: O(1)
  • Goed voor dense graphs

Adjacency Lijst:

  • Array van lijsten
  • Geheugen: O(V + E)
  • Check edge: O(degree)
  • Goed voor sparse graphs

Belangrijkste inzichten

  • Graphs = vertices + edges
  • DFS: Stack, go deep
  • BFS: Queue, go wide
  • BFS finds kortste pad (unweighted)
  • Beide O(V + E) complexiteit
  • Track visited naar avoid infinite lussen
