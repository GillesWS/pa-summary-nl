What are Graphs?

Definition: Non-linear data structure with vertices (nodes) and edges (connections)

Components:

  • Vertices/Nodes: Points in graph
  • Edges: Connections between vertices

Types:

  • Directed: One-way edges
  • Undirected: Two-way edges
  • Weighted: Edges have costs
  • Unweighted: All edges equal

A lot of different graphs

Types of Graphs:

  • Undirected: Relationships are mutual (eg roads between cities)
  • Directed: Edges indicate a direction (like parents/children)
  • Disconnected: One or more nodes are not connected (like continents)
  • Acyclic: No cycles
  • Complete: Every node is connected to every other node
  • Weighted: Edges between nodes have a weight

DFS - Depth First Search

Strategy: Go deep, explore one path completely before backtracking

Implementation:

  • Use Stack (or recursion)
  • Track visited nodes
  • LIFO order
DFS(node, visited): mark node as visited FOR each neighbor of node: IF neighbor not visited: DFS(neighbor, visited)

Complexity: O(V + E) where V=vertices, E=edges

BFS - Breadth First Search

Strategy: Go wide, explore all neighbors before moving deeper

Implementation:

  • Use Queue
  • Track visited nodes
  • FIFO order
BFS(start): queue = [start] mark start as visited WHILE queue not empty: node = dequeue() FOR each neighbor: IF not visited: mark visited enqueue(neighbor)

Complexity: O(V + E)

🆚 DFS vs BFS

DFS (Stack/Recursion):

  • + Less memory for deep graphs
  • + Good for detecting cycles
  • - Doesn't guarantee shortest path

BFS (Queue):

  • + Finds shortest path (unweighted)
  • + Level-by-level exploration
  • - More memory for wide graphs

Graph Representations

Adjacency Matrix:

  • 2D array [V][V]
  • Space: O(V²)
  • Check edge: O(1)
  • Good for dense graphs

Adjacency List:

  • Array of lists
  • Space: O(V + E)
  • Check edge: O(degree)
  • Good for sparse graphs

Key Takeaways

  • Graphs = vertices + edges
  • DFS: Stack, go deep
  • BFS: Queue, go wide
  • BFS finds shortest path (unweighted)
  • Both O(V + E) complexity
  • Track visited to avoid infinite loops
