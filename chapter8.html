Wat zijn grafen?

Definitie: Niet-lineaire datastructuur met vertices (knopen) en edges (verbindingen)

Onderdelen:

  • Vertices/knopen: Punten in de graaf
  • Edges/verbindingen: Verbindingen tussen vertices

Types:

  • Gericht: Verbindingen in één richting
  • Ongericht: Verbindingen in twee richtingen
  • Gewogen: Verbindingen hebben kosten
  • Ongewogen: Alle verbindingen zijn gelijk

Er bestaan veel verschillende grafen

Types grafen:

  • Ongericht: Relaties zijn wederzijds (bv. wegen tussen steden)
  • Gericht: Verbindingen geven een richting aan (zoals ouders/kinderen)
  • Niet-verbonden: Eén of meer knopen zijn niet verbonden (zoals continenten)
  • Acyclisch: Geen cycli
  • Compleet: Elke knoop is verbonden met elke andere knoop
  • Gewogen: Verbindingen tussen knopen hebben een gewicht

DFS - Depth-first search

Strategie: Ga diep: verken één pad volledig voordat je backtrackt

Implementation:

  • Gebruik een stack (of recursie)
  • Houd bezochte knopen bij
  • LIFO-volgorde
DFS(node, visited): markeer knoop as bezocht FOR each neighbor of node: IF neighbor not visited: DFS(neighbor, visited)

Complexiteit: O(V + E) where V=vertices, E=edges

BFS - Breadth-first search

Strategie: Ga breed: verken alle buren voordat je dieper gaat

Implementation:

  • Gebruik een wachtrij
  • Houd bezochte knopen bij
  • FIFO-volgorde
BFS(start): wachtrij = [start] markeer start as bezocht WHILE queue not empty: node = dequeue() FOR each neighbor: IF not visited: markeer bezocht enqueue(neighbor)

Complexiteit: O(V + E)

🆚 DFS vs BFS

DFS (stack/recursie):

  • + Minder geheugen voor diepe grafen
  • + Goed om cycli te detecteren
  • - Garandeert geen kortste pad

BFS (wachtrij):

  • + Vindt het kortste pad (ongewogen)
  • + Verkenning niveau per niveau
  • - Meer geheugen voor brede grafen

Graafrepresentaties

Adjacentiematrix:

  • 2D-array [V][V]
  • Ruimte: O(V²)
  • Controleer verbinding: O(1)
  • Goed voor dense grafen

Adjacentielijst:

  • Array van lijsten
  • Ruimte: O(V + E)
  • Controleer verbinding: O(graad)
  • Goed voor sparse grafen

Belangrijkste inzichten

  • Grafen = vertices + edges
  • DFS: stack, ga diep
  • BFS: wachtrij, ga breed
  • BFS vindt het kortste pad (ongewogen)
  • Beide hebben O(V + E)-complexiteit
  • Houd visited bij om oneindige lussen te vermijden
