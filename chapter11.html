Wat is een boom?

Definitie: Een datastructuur verwant aan een graaf

Boomterminologie:

  • Root-knoop: Bovenste knoop (start de boom)
  • Ouder: Knoop met kinderen
  • Blad: Knoop zonder kinderen
  • Broers/zussen: Knopen met dezelfde ouder
  • Hoogte: Aantal verbindingen van root naar diepste blad
  • Diepte: Aantal verbindingen van knoop naar root

Types bomen

Volledige binaire boom:

Elke knoop heeft 0 of 2 kinderen (nooit 1)

Complete binaire boom:

Alle niveaus gevuld behalve mogelijk het laatste, gevuld van links naar rechts

Perfecte binaire boom:

Alle interne knopen hebben 2 kinderen, alle bladeren zitten op hetzelfde niveau

Gebalanceerde boom:

Hoogteverschil tussen linker- en rechtersubboom ≤ 1

Gedegenereerde boom:

Elke knoop heeft slechts 1 kind (zoals een linked list)

Gedegenereerde bomen hebben O(n)-bewerkingen - slechtste geval!

Bomen in de praktijk

Bestandssystemen:

Mappen en bestanden hiërarchisch georganiseerd

Organigrammen:

Bedrijfshiërarchie met CEO aan de root

Beslissingsbomen:

AI/ML voor classificatie en beslissingen

DOM (HTML):

Webpagina’s gestructureerd als boom van elementen

Stambomen:

Genealogie en afkomst bijhouden

Expressiebomen:

Wiskundige expressies: (3+5)*2

Binaire zoekboom (BST)

Definitie: Binaire boom met ordeningseigenschap

BST-eigenschap:

  • Linkersubboom: Alle waarden < knoop
  • Rechtersubboom: Alle waarden > knoop
  • Beide subbomen zijn ook BST’s

Structuur:

class Node { int Value; Node Left; Node Right; }

Toevoegen aan BST

Recursieve aanpak:

Add(node, value): IF node is null: RETURN new Node(value) IF value < node.Value: node.Left = Add(node.Left, value) ELSE: node.Right = Add(node.Right, value) RETURN node

Complexiteit: O(h) waarbij h = hoogte
Beste: O(log n), slechtste: O(n)

Zoeken in BST

Search(node, value): IF node is null: RETURN false IF value == node.Value: RETURN true IF value < node.Value: RETURN Search(node.Left, value) ELSE: RETURN Search(node.Right, value)

Complexiteit: O(h) - hetzelfde als toevoegen

Boomtraversals

In-order (links → root → rechts):

Bezoekt knopen in oplopende volgorde

Voorbeeld boom: 2, 3, 4, 5, 6, 7, 8

Gebruik: Gesorteerde data krijgen

Pre-order (root → links → rechts):

Bezoekt ouder vóór kinderen

Voorbeeld boom: 5, 3, 2, 4, 7, 6, 8

Gebruik: Bomen kopiëren/klonen

Post-order (links → rechts → root):

Bezoekt kinderen vóór ouder

Voorbeeld boom: 2, 4, 3, 6, 8, 7, 5

Gebruik: Bomen veilig verwijderen

Implementaties van traversals

InOrder(node): IF node is null: RETURN InOrder(node.Left) Print(node.Value) InOrder(node.Right) PreOrder(node): IF node is null: RETURN Print(node.Value) PreOrder(node.Left) PreOrder(node.Right) PostOrder(node): IF node is null: RETURN PostOrder(node.Left) PostOrder(node.Right) Print(node.Value)

Wanneer gebruik je welke traversal?

In-Order:

  • Data in gesorteerde volgorde krijgen
  • BST-eigenschap valideren

Pre-Order:

  • Bomen kopiëren/klonen
  • Boom maken vanuit traversal
  • Ouder moet vóór kinderen komen

Post-Order:

  • Bomen verwijderen (verwijder kinderen eerst)
  • Expressiebomen evalueren
  • Kinderen moeten vóór ouder komen

Min en max

Minimum zoeken:

FindMin(node): WHILE node.Left != null: node = node.Left RETURN node.Value

Ga helemaal naar links!

Maximum zoeken:

FindMax(node): WHILE node.Right != null: node = node.Right RETURN node.Value

Ga helemaal naar rechts!

Belangrijkste inzichten

  • Boom = datastructuur verwant aan een graaf
  • Root = bovenste knoop, blad = geen kinderen, ouder = heeft kinderen
  • Hoogte = verbindingen van root naar diepste blad
  • Diepte = verbindingen van knoop naar root
  • Gedegenereerde boom = elke knoop heeft 1 kind (slechtste geval)
  • BST: links < knoop < rechts
  • Bewerkingen: O(h)-complexiteit
  • Gebalanceerd: O(log n), ongebalanceerd: O(n)
  • In-order → gesorteerde uitvoer
  • Pre-order → ouder eerst (kopiëren)
  • Post-order → kinderen eerst (verwijderen)
  • Min = meest links, max = meest rechts
  • Praktijk: bestandssystemen, organigrammen, HTML DOM
