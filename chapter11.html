Wat is een Tree?

Definitie: Een data structure related naar een graph

Tree Terminology:

  • Root node: Top node (starts de tree)
  • Parent: Node met kinderen
  • Leaf: Node zonder kinderen
  • Siblings: Nodes met dezelfde parent
  • Height: Number van edges van root naar deepest leaf
  • Depth: Number van edges van node naar root

Types van Trees

Full Binary Tree:

Elke node has 0 of 2 kinderen (nooit 1)

Complete Binary Tree:

Alle levels filled except possibly last, filled links naar rechts

Perfect Binary Tree:

Alle internal nodes have 2 kinderen, alle leaves at dezelfde level

Balanced Tree:

Height difference between links en rechts subtrees ≤ 1

Degenerate Tree:

Elke node has alleen 1 child (like een linked lijst)

Degenerate trees have O(n) bewerkingen - slechtste case!

Trees in Real Life

File Systems:

Folders en files organized hierarchically

Organization Charts:

Company hierarchy met CEO at root

Decision Trees:

AI/ML voor classification en decisions

DOM (HTML):

Web pages structured als tree van elementen

Family Trees:

Genealogy en ancestry tracking

Expression Trees:

Mathematical expressions: (3+5)*2

Binary Zoeken Tree (BST)

Definitie: Binary tree met ordering property

BST Property:

  • Links subtree: Alle waarden < node
  • Rechts subtree: Alle waarden > node
  • Beide subtrees zijn also BSTs

Structure:

les Node { int Waarde; Node Links; Node Rechts; }

Adding naar BST

Recursief approach:

Add(node, waarde): IF node is null: RETURN new Node(waarde) IF waarde < node.Waarde: node.Links = Add(node.Links, waarde) ELSE: node.Rechts = Add(node.Rechts, waarde) RETURN node

Complexiteit: O(h) where h = height
Best: O(log n), Worst: O(n)

Searching BST

Zoeken(node, waarde): IF node is null: RETURN niet waar IF waarde == node.Waarde: RETURN waar IF waarde < node.Waarde: RETURN Zoeken(node.Links, waarde) ELSE: RETURN Zoeken(node.Rechts, waarde)

Complexiteit: O(h) - dezelfde als add

Tree Traversals

In-Order (Links → Root → Rechts):

Visits nodes in ascending volgorde

Example tree: 2, 3, 4, 5, 6, 7, 8

Gebruik: Get gesorteerde data

Pre-Order (Root → Links → Rechts):

Visits parent before kinderen

Example tree: 5, 3, 2, 4, 7, 6, 8

Gebruik: Copy/clone trees

Post-Order (Links → Rechts → Root):

Visits kinderen before parent

Example tree: 2, 4, 3, 6, 8, 7, 5

Gebruik: Verwijderen trees safely

Traversal Implementations

InOrder(node): IF node is null: RETURN InOrder(node.Links) Print(node.Waarde) InOrder(node.Rechts) PreOrder(node): IF node is null: RETURN Print(node.Waarde) PreOrder(node.Links) PreOrder(node.Rechts) PostOrder(node): IF node is null: RETURN PostOrder(node.Links) PostOrder(node.Rechts) Print(node.Waarde)

Wanneer naar Gebruik Welke Traversal

In-Order:

  • Getting data in gesorteerd volgorde
  • Validating BST property

Pre-Order:

  • Copying/cloning trees
  • Creating tree van traversal
  • Nodig hebben parent before kinderen

Post-Order:

  • Deleting trees (verwijderen kinderen first)
  • Evaluating expression trees
  • Nodig hebben kinderen before parent

Min en Max

Find Minimum:

FindMin(node): WHILE node.Links != null: node = node.Links RETURN node.Waarde

Go alle de way links!

Find Maximum:

FindMax(node): WHILE node.Rechts != null: node = node.Rechts RETURN node.Waarde

Go alle de way rechts!

Belangrijkste inzichten

  • Tree = data structure related naar graph
  • Root = top node, Leaf = no kinderen, Parent = has kinderen
  • Height = edges van root naar deepest leaf
  • Depth = edges van node naar root
  • Degenerate tree = elke node has 1 child (slechtste case)
  • BST: links < node < rechts
  • Bewerkingen: O(h) complexiteit
  • Balanced: O(log n), Unbalanced: O(n)
  • In-Order → gesorteerd output
  • Pre-Order → parent first (copy)
  • Post-Order → kinderen first (verwijderen)
  • Min = leftmost, Max = rightmost
  • Real-life: file systems, org charts, HTML DOM
