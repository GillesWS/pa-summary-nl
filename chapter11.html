Binary trees, BST en traversals

Samenvatting

Een tree is een speciale graph zonder cycles, met een hiërarchische structuur vanaf een root.

Een binary tree heeft maximaal twee kinderen per node. Een binary search tree plaatst kleinere waarden links en grotere waarden rechts.

Traversals bepalen in welke volgorde je nodes bezoekt: in-order, pre-order, post-order of level-order.

Wat moet je begrijpen?

  • Root: bovenste node.
  • Leaf: node zonder kinderen.
  • Height: langste pad naar een leaf.
  • BST property: links kleiner, rechts groter.
  • In-order traversal geeft sorted output bij BST.

Big O & complexiteit

  • BST search gemiddeld: O(log n).
  • BST search slechtste geval: O(n).
  • Traversal van alle nodes: O(n).
  • Min/max in BST: O(h), waarbij h de hoogte is.

C# voorbeeld

static void InOrder(Node node)
{
    if (node == null) return;

    InOrder(node.Left);
    Console.WriteLine(node.Value);
    InOrder(node.Right);
}

Typische valkuilen

  • Denken dat een BST altijd O(log n) is.
  • In-order, pre-order en post-order verwarren.
  • Null-checks vergeten.
  • Sorted input invoegen waardoor de tree kan degenereren.
