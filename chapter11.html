What is a Tree?

Definition: A data structure related to a graph

Tree Terminology:

  • Root node: Top node (starts the tree)
  • Parent: Node with children
  • Leaf: Node without children
  • Siblings: Nodes with same parent
  • Height: Number of edges from root to deepest leaf
  • Depth: Number of edges from node to root

Types of Trees

Full Binary Tree:

Every node has 0 or 2 children (never 1)

Complete Binary Tree:

All levels filled except possibly last, filled left to right

Perfect Binary Tree:

All internal nodes have 2 children, all leaves at same level

Balanced Tree:

Height difference between left and right subtrees ≤ 1

Degenerate Tree:

Each node has only 1 child (like a linked list)

Degenerate trees have O(n) operations - worst case!

Trees in Real Life

File Systems:

Folders and files organized hierarchically

Organization Charts:

Company hierarchy with CEO at root

Decision Trees:

AI/ML for classification and decisions

DOM (HTML):

Web pages structured as tree of elements

Family Trees:

Genealogy and ancestry tracking

Expression Trees:

Mathematical expressions: (3+5)*2

Binary Search Tree (BST)

Definition: Binary tree with ordering property

BST Property:

  • Left subtree: All values < node
  • Right subtree: All values > node
  • Both subtrees are also BSTs

Structure:

class Node { int Value; Node Left; Node Right; }

Adding to BST

Recursive approach:

Add(node, value): IF node is null: RETURN new Node(value) IF value < node.Value: node.Left = Add(node.Left, value) ELSE: node.Right = Add(node.Right, value) RETURN node

Complexity: O(h) where h = height
Best: O(log n), Worst: O(n)

Searching BST

Search(node, value): IF node is null: RETURN false IF value == node.Value: RETURN true IF value < node.Value: RETURN Search(node.Left, value) ELSE: RETURN Search(node.Right, value)

Complexity: O(h) - same as add

Tree Traversals

In-Order (Left → Root → Right):

Visits nodes in ascending order

Example tree: 2, 3, 4, 5, 6, 7, 8

Use: Get sorted data

Pre-Order (Root → Left → Right):

Visits parent before children

Example tree: 5, 3, 2, 4, 7, 6, 8

Use: Copy/clone trees

Post-Order (Left → Right → Root):

Visits children before parent

Example tree: 2, 4, 3, 6, 8, 7, 5

Use: Delete trees safely

Traversal Implementations

InOrder(node): IF node is null: RETURN InOrder(node.Left) Print(node.Value) InOrder(node.Right) PreOrder(node): IF node is null: RETURN Print(node.Value) PreOrder(node.Left) PreOrder(node.Right) PostOrder(node): IF node is null: RETURN PostOrder(node.Left) PostOrder(node.Right) Print(node.Value)

When to Use Which Traversal

In-Order:

  • Getting data in sorted order
  • Validating BST property

Pre-Order:

  • Copying/cloning trees
  • Creating tree from traversal
  • Need parent before children

Post-Order:

  • Deleting trees (delete children first)
  • Evaluating expression trees
  • Need children before parent

Min and Max

Find Minimum:

FindMin(node): WHILE node.Left != null: node = node.Left RETURN node.Value

Go all the way left!

Find Maximum:

FindMax(node): WHILE node.Right != null: node = node.Right RETURN node.Value

Go all the way right!

Key Takeaways

  • Tree = data structure related to graph
  • Root = top node, Leaf = no children, Parent = has children
  • Height = edges from root to deepest leaf
  • Depth = edges from node to root
  • Degenerate tree = each node has 1 child (worst case)
  • BST: left < node < right
  • Operations: O(h) complexity
  • Balanced: O(log n), Unbalanced: O(n)
  • In-Order → sorted output
  • Pre-Order → parent first (copy)
  • Post-Order → children first (delete)
  • Min = leftmost, Max = rightmost
  • Real-life: file systems, org charts, HTML DOM
