Chapter 5: Divide & Conquer
← Back to Chapters

️ What is Divide & Conquer?

Definition: Algorithm design paradigm that breaks problems into smaller subproblems, solves them recursively, and combines results.

Three Steps:

  • 1. Divide: Break problem into smaller subproblems
  • 2. Conquer: Solve each subproblem recursively
  • 3. Combine: Merge solutions to get final result
Key Insight: Transform big problems into manageable pieces!

Quick Sort

Strategy: Pick pivot, partition around it, recursively sort partitions

Algorithm Steps:

  • 1. Pick pivot: Any element (first, last, random, median)
  • 2. Partition: Elements < pivot | pivot | elements > pivot
  • 3. Recursively sort: Both partitions
  • 4. Combine: [sorted less] + [pivot] + [sorted greater]

Base Case:

Arrays with 0 or 1 element (already sorted)

Complexity:

  • Best/Average: O(n log n)
  • Worst: O(n²) - poor pivot choices
  • Space: O(log n) - recursion stack

Quick Sort Example

Array: [12, 56, 34, 78, 23] 1. Pivot = 12 Less: [] Greater: [56, 34, 78, 23] 2. Sort Greater (pivot = 56): Less: [34, 23] Greater: [78] 3. Continue recursively... Final: [12, 23, 34, 56, 78]

Tower of Hanoi

Problem: Move stack of disks from source to destination following rules.

Rules:

  • Only one disk moved at a time
  • No disk on top of smaller disk
  • Use auxiliary peg as needed

Recursive Strategy:

To move n disks from A to C using B: 1. Move n-1 disks from A to B (using C) 2. Move disk n from A to C 3. Move n-1 disks from B to C (using A)

Base Case:

n = 1: Simply move disk from source to destination

Elegance: Complex problem solved with simple recursive pattern!

D&C Time Complexity

Typical Pattern: O(n log n)

Why O(n log n)?

  • Divide: Split in half → log n levels
  • Work per level: O(n) total work
  • Total: n × log n = O(n log n)

Performance Comparison (200 elements):

Bubble Sort: 19,900 operations Selection Sort: 19,900 operations Insertion Sort: 11,037 operations Quick Sort: 1,999 operations ⚡

Classic D&C Examples

1. Merge Sort:

Split array, sort halves, merge back together. Work happens AFTER recursion.

2. Quick Sort:

Partition around pivot, sort partitions. Work happens BEFORE recursion.

3. Binary Search:

Split sorted array in half, search relevant half. O(log n)

4. Tower of Hanoi:

Move n-1 disks, move disk n, move n-1 disks again.

5. Array Multiplication:

Split arrays, multiply parts, combine results.

Divide vs Conquer

Divide Phase:

  • Breaking problem into subproblems
  • Usually O(1) or O(n) work
  • Example: Finding midpoint, partitioning

Conquer Phase:

  • Solving subproblems recursively
  • Keeps dividing until base case
  • Where recursion happens

Combine Phase:

  • Merging solutions together
  • May be trivial or complex
  • Example: Merging sorted arrays

Key Takeaways

  • Three steps: Divide, Conquer, Combine
  • Breaks big problems into smaller pieces
  • Uses recursion to solve subproblems
  • Typical complexity: O(n log n)
  • Quick Sort: partitions then recurses
  • Merge Sort: recurses then merges
  • Binary Search: simplest D&C example
  • Tower of Hanoi: elegant recursive solution
  • Much faster than naive O(n²) algorithms!
