️ Wat is Divide & Conquer?

Definitie: Algoritme design paradigm dat breaks problemen into kleiner subproblemen, solves them recursively, en combines results.

Three Stappen:

  • 1. Divide: Break probleem into kleiner subproblemen
  • 2. Conquer: Oplossen elke subprobleem recursively
  • 3. Combine: Merge oplossingen naar get final result
Key Insight: Transform big problemen into manageable pieces!

Quick sort

Strategie: Pick pivot, partition around it, recursively sort partitions

Algoritme Stappen:

  • 1. Pick pivot: Any element (first, last, random, median)
  • 2. Partition: Elementen < pivot | pivot | elementen > pivot
  • 3. Recursively sort: Beide partitions
  • 4. Combine: [gesorteerd less] + [pivot] + [gesorteerd greater]

Base Case:

Arrays met 0 of 1 element (already gesorteerd)

Complexiteit:

  • Best/Average: O(n log n)
  • Worst: O(n²) - poor pivot keuzes
  • Geheugen: O(log n) - recursie stack

Quick Sort Example

Array: [12, 56, 34, 78, 23] 1. Pivot = 12 Less: [] Greater: [56, 34, 78, 23] 2. Sort Greater (pivot = 56): Less: [34, 23] Greater: [78] 3. Continue recursively... Final: [12, 23, 34, 56, 78]

Tower van Hanoi

Probleem: Move stack van disks van source naar destination following rules.

Rules:

  • Only one disk moved at een tijd
  • Nee disk on top van kleiner disk
  • Gebruik auxiliary peg als needed

Recursief Strategie:

Naar move n disks van Een naar C gebruik van B: 1. Move n-1 disks van Een naar B (gebruik van C) 2. Move disk n van Een naar C 3. Move n-1 disks van B naar C (gebruik van Een)

Base Case:

n = 1: Simply move disk van source naar destination

Elegance: Complex probleem solved met simple recursief patroon!

D&C Tijd Complexiteit

Typical Patroon: O(n log n)

Waarom O(n log n)?

  • Divide: Split in half → log n levels
  • Werkt per level: O(n) total werkt
  • Total: n × log n = O(n log n)

Performance Vergelijking (200 elementen):

Bubble Sort: 19,900 bewerkingen Selection Sort: 19,900 bewerkingen Insertion Sort: 11,037 bewerkingen Quick Sort: 1,999 bewerkingen ⚡

Classic D&C Examples

1. Merge Sort:

Split array, sort halves, merge back together. Werkt happens AFTER recursie.

2. Quick Sort:

Partition around pivot, sort partitions. Werkt happens BEFORE recursie.

3. Binary Zoeken:

Split gesorteerd array in half, zoeken relevant half. O(log n)

4. Tower van Hanoi:

Move n-1 disks, move disk n, move n-1 disks again.

5. Array Multiplication:

Split arrays, multiply parts, combine results.

Divide vs Conquer

Divide Phase:

  • Breaking probleem into subproblemen
  • Usually O(1) of O(n) werkt
  • Voorbeeld: Finding midpoint, partitioning

Conquer Phase:

  • Solving subproblemen recursively
  • Keeps dividing until base case
  • Where recursie happens

Combine Phase:

  • Merging oplossingen together
  • May be trivial of complex
  • Voorbeeld: Merging gesorteerd arrays

Belangrijkste inzichten

  • Three stappen: Divide, Conquer, Combine
  • Breaks big problemen into kleiner pieces
  • Gebruikt recursie naar oplossen subproblemen
  • Typical complexiteit: O(n log n)
  • Quick Sort: partitions then recurses
  • Merge Sort: recurses then merges
  • Binary Zoeken: simplest D&C example
  • Tower van Hanoi: elegant recursief oplossing
  • Much sneller dan naive O(n²) algoritmes!
