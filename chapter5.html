05. Divide & Conquer

Divide, conquer, combine

Samenvatting

Divide & Conquer splitst een probleem in kleinere gelijkaardige subproblemen, lost die apart op en combineert de resultaten.

Bekende voorbeelden zijn binary search, merge sort, quick sort en Tower of Hanoi.

Het verschil met gewone recursie is dat divide & conquer vaak meerdere recursieve calls maakt en daarna resultaten samenbrengt.

Wat moet je begrijpen?

  • Divide: splits het probleem.
  • Conquer: los subproblemen op.
  • Combine: voeg resultaten samen.
  • Quicksort doet partitionering vóór recursie; merge sort combineert na recursie.

Big O & complexiteit

  • Binary search: O(log n).
  • Merge sort: O(n log n).
  • Quicksort gemiddeld O(n log n), worst-case O(n²).
  • Tower of Hanoi: O(2^n).

C# voorbeeld

static int SumDivide(int[] a, int left, int right)
{
    if (left > right) return 0;
    if (left == right) return a[left];

    int mid = (left + right) / 2;
    return SumDivide(a, left, mid) + SumDivide(a, mid + 1, right);
}

Typische valkuilen

  • Combine-stap vergeten.
  • D&C gebruiken terwijl subproblemen sterk overlappen; dan is DP vaak beter.
  • Geen duidelijke base case hebben.
  • Quicksort altijd O(n log n) noemen zonder worst-case te vermelden.
