10. Greedy Algorithms

Lokale keuzes en global optimum

Samenvatting

Greedy bouwt een oplossing stap voor stap en kiest telkens wat op dat moment het beste lijkt.

Greedy is snel en eenvoudig, maar werkt alleen optimaal als de greedy choice property geldig is.

Activity selection is een klassiek voorbeeld waar greedy werkt: sorteer op eindtijd en kies telkens de activiteit die het vroegst eindigt.

Wat moet je begrijpen?

  • Local optimum: beste keuze nu.
  • Global optimum: beste totale oplossing.
  • Greedy choice property: lokale keuze kan deel zijn van optimale oplossing.
  • Optimal substructure: optimale oplossing bestaat uit optimale deeloplossingen.

Big O & complexiteit

  • Vaak O(n log n) door sorteren.
  • Na sorteren vaak O(n) voor selectie.
  • Greedy gebruikt vaak weinig extra geheugen: O(1) of O(n) afhankelijk van output.

C# voorbeeld

// Activity selection idee:
activities.Sort((a, b) => a.End.CompareTo(b.End));

int lastEnd = int.MinValue;
foreach (var act in activities)
{
    if (act.Start >= lastEnd)
    {
        Console.WriteLine(act.Name);
        lastEnd = act.End;
    }
}

Typische valkuilen

  • Aannemen dat greedy altijd optimaal is.
  • Verkeerd sorteercriterium kiezen.
  • Greedy gebruiken bij 0/1 knapsack en denken dat het altijd optimaal is.
  • Geen tegenvoorbeeld kunnen geven.
