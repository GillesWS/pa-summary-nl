What zijn Data Structures?

Definitie: Ways naar organize en opslaan data voor efficiënt toegang en modification.

Waarom Learn Them?

  • Understand fundamentals: Better tool usage
  • Know when naar gebruik what: Rechts structure voor rechts job
  • Performance awareness: Waarom some bewerkingen zijn fast/slow
  • Debug effectively: Know what's happening under de hood
Real-world: Gebruik built-in libraries (Stack, Queue, Lijst). Don't reinvent de wheel!

Arrays

Structure: Fixed-size, contiguous geheugen blocks

Complexiteit:

  • Toegang door index: O(1)
  • Zoeken: O(n)
  • Insert/Delete: O(n) - vereist shifting

Characteristics:

  • Fixed grootte (must know grootte at creation)
  • Snel: random toegang
  • Elementen stored contiguously
  • Cache-friendly

Linked Lijsten

Structure: Nodes connected door references/pointers

Node Components:

  • Data: De waarde stored
  • Next: Reference naar next node

Complexiteit:

  • Toegang: O(n) - must traverse van head
  • Zoeken: O(n)
  • Invoegen at head: O(1)
  • Verwijderen (if have reference): O(1)

Types:

  • Singly: One-way references
  • Doubly: Two-way references (prev & next)
  • Circular: Last node links naar first
Advantage: Dynamic grootte - grows/shrinks als needed!

Stacks (LIFO)

Principle: Last In, First Out

Key Bewerkingen:

  • Push: Add element naar top - O(1)
  • Pop: Remove element van top - O(1)
  • Peek: View top element - O(1)

Implementation:

Can gebruik array of linked lijst

Important Concepts:

  • Stack Overflow: Push when stack is full
  • Stack Underflow: Pop when stack is empty

Real-Life Examples:

  • Browser back button
  • Undo bewerkingen
  • Functie call stack
  • Backtracking algoritmes

Queues (FIFO)

Principle: First In, First Out

Key Bewerkingen:

  • Enqueue: Add naar end - O(1)
  • Dequeue: Remove van front - O(1)
  • Peek: View front element - O(1)

Implementation:

Can gebruik array of linked lijst

Real-Life Examples:

  • Printer queue
  • Customer service line
  • Task scheduling
  • Breadth-first zoeken

Stack vs Queue

Stack (LIFO):

  • Like een stack van plates - take van top
  • Bewerkingen: push, pop, peek
  • Gebruik when: volgorde matters (undo, recursie)

Queue (FIFO):

  • Like een line at opslaan - first person first served
  • Bewerkingen: enqueue, dequeue, peek
  • Gebruik when: fairness matters (printing, scheduling)

Belangrijkste inzichten

  • Arrays: Snel: toegang O(1), fixed grootte
  • Linked Lijsten: Dynamic grootte, O(n) toegang
  • Stacks: LIFO - last in, first out
  • Queues: FIFO - first in, first out
  • Choose based on your heeft nodig (toegang patronen, grootte changes)
  • Gebruik built-in libraries in production
  • Understanding fundamentals helps debug en optimize
