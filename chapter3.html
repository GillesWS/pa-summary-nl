Arrays, linked lists, stacks en queues

Samenvatting

Een datastructuur bepaalt hoe data wordt opgeslagen en hoe snel je bepaalde operaties kunt uitvoeren.

Arrays zijn sterk voor snelle index-access. Linked lists zijn flexibeler voor invoegen en verwijderen wanneer je de juiste node hebt. Stacks gebruiken LIFO, queues gebruiken FIFO.

De juiste datastructuur kiezen is vaak belangrijker dan achteraf proberen je code sneller te maken.

Wat moet je begrijpen?

  • Array: vaste/contigue opslag, access O(1), search O(n).
  • Linked list: nodes met verwijzingen, access/search O(n).
  • Stack: Last In First Out, handig voor undo, recursion en DFS.
  • Queue: First In First Out, handig voor BFS en wachtrijen.

Big O & complexiteit

  • Array access: O(1)
  • Array search: O(n)
  • Linked list access/search: O(n)
  • Stack push/pop: O(1)
  • Queue enqueue/dequeue: O(1)

C# voorbeeld

var stack = new Stack<string>();
stack.Push("eerste");
stack.Push("tweede");
Console.WriteLine(stack.Pop()); // tweede

var queue = new Queue<string>();
queue.Enqueue("eerste");
queue.Enqueue("tweede");
Console.WriteLine(queue.Dequeue()); // eerste

Typische valkuilen

  • Stack en queue verwarren.
  • Denken dat linked lists altijd sneller zijn.
  • O(1) insert/delete claimen zonder dat je de node al hebt.
  • Array insert/delete vergeten dat shifting nodig kan zijn.
