Hoofdstuk 3: Datastructuren
← Terug naar hoofdstukken

Wat zijn datastructuren?

Definitie: Manieren om data te organiseren en op te slaan voor efficiënte toegang en aanpassing.

Waarom leer je ze?

  • Begrijp de basis: Beter gebruik van tools
  • Weet wanneer je wat moet gebruiken: De juiste structuur voor de juiste taak
  • Prestatiebewustzijn: Waarom sommige bewerkingen snel/traag zijn
  • Effectief debuggen: Weten wat er achter de schermen gebeurt
Praktijk: Gebruik ingebouwde bibliotheken (Stack, Wachtrij, Lijst). Vind het wiel niet opnieuw uit!

Arrays

Structuur: Vaste grootte, aaneengesloten geheugenblokken

Complexiteit:

  • Toegang via index: O(1)
  • Zoeken: O(n)
  • Invoegen/verwijderen: O(n) - vereist verschuiven

Kenmerken:

  • Vaste grootte (je moet de grootte kennen bij aanmaak)
  • Snelle willekeurige toegang
  • Elementen worden aaneengesloten opgeslagen
  • Cachevriendelijk

Linked lists

Structuur: Knopen verbonden door referenties/pointers

Onderdelen van een knoop:

  • Data: De opgeslagen waarde
  • Next: Referentie naar de volgende knoop

Complexiteit:

  • Toegang: O(n) - moet vanaf de head doorlopen
  • Zoeken: O(n)
  • Invoegen aan de head: O(1)
  • Verwijderen (als je de referentie hebt): O(1)

Types:

  • Singly: Referenties in één richting
  • Doubly: Referenties in twee richtingen (prev & next)
  • Circular: De laatste knoop verwijst naar de eerste
Voordeel: Dynamische grootte - groeit/krimpt wanneer nodig!

Stacks (LIFO)

Principle: Last In, First Out

Belangrijke bewerkingen:

  • Push: Voeg element bovenaan toe - O(1)
  • Pop: Verwijder element bovenaan - O(1)
  • Peek: Bekijk het bovenste element - O(1)

Implementation:

Kan array of linked list gebruiken

Belangrijke concepten:

  • Stack overflow: Push wanneer de stack vol is
  • Stack underflow: Pop wanneer de stack leeg is

Praktijk Voorbeelden:

  • Terugknop van de browser
  • Undo-bewerkingen
  • Function call stack
  • Backtracking-algoritmen

Wachtrijen (FIFO)

Principle: First In, First Out

Belangrijke bewerkingen:

  • Enqueue: Voeg achteraan toe - O(1)
  • Dequeue: Verwijder vooraan - O(1)
  • Peek: Bekijk het voorste element - O(1)

Implementation:

Kan array of linked list gebruiken

Praktijk Voorbeelden:

  • Printerwachtrij
  • Wachtrij bij klantendienst
  • Taakplanning
  • Breadth-first search

Stack vs wachtrij

Stack (LIFO):

  • Zoals een stapel borden - je neemt van bovenaan
  • Bewerkingen: push, pop, peek
  • Gebruik wanneer volgorde belangrijk is (undo, recursie)

Queue (FIFO):

  • Zoals een wachtrij in de winkel - wie eerst komt, wordt eerst geholpen
  • Bewerkingen: enqueue, dequeue, peek
  • Gebruik wanneer eerlijkheid belangrijk is (printen, planning)

Belangrijkste inzichten

  • Arrays: snelle toegang O(1), vaste grootte
  • Linked lists: dynamische grootte, O(n) toegang
  • Stacks: LIFO - last in, first out
  • Wachtrijen: FIFO - first in, first out
  • Kies op basis van je noden (toegangspatronen, groottewijzigingen)
  • Gebruik ingebouwde bibliotheken in productie
  • De basis begrijpen helpt bij debuggen en optimaliseren
