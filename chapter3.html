What are Data Structures?

Definition: Ways to organize and store data for efficient access and modification.

Why Learn Them?

  • Understand fundamentals: Better tool usage
  • Know when to use what: Right structure for right job
  • Performance awareness: Why some operations are fast/slow
  • Debug effectively: Know what's happening under the hood
Real-world: Use built-in libraries (Stack, Queue, List). Don't reinvent the wheel!

Arrays

Structure: Fixed-size, contiguous memory blocks

Complexity:

  • Access by index: O(1)
  • Search: O(n)
  • Insert/Delete: O(n) - requires shifting

Characteristics:

  • Fixed size (must know size at creation)
  • Fast random access
  • Elements stored contiguously
  • Cache-friendly

Linked Lists

Structure: Nodes connected by references/pointers

Node Components:

  • Data: The value stored
  • Next: Reference to next node

Complexity:

  • Access: O(n) - must traverse from head
  • Search: O(n)
  • Insert at head: O(1)
  • Delete (if have reference): O(1)

Types:

  • Singly: One-way references
  • Doubly: Two-way references (prev & next)
  • Circular: Last node links to first
Advantage: Dynamic size - grows/shrinks as needed!

Stacks (LIFO)

Principle: Last In, First Out

Key Operations:

  • Push: Add element to top - O(1)
  • Pop: Remove element from top - O(1)
  • Peek: View top element - O(1)

Implementation:

Can use array or linked list

Important Concepts:

  • Stack Overflow: Push when stack is full
  • Stack Underflow: Pop when stack is empty

Real-Life Examples:

  • Browser back button
  • Undo operations
  • Function call stack
  • Backtracking algorithms

Queues (FIFO)

Principle: First In, First Out

Key Operations:

  • Enqueue: Add to end - O(1)
  • Dequeue: Remove from front - O(1)
  • Peek: View front element - O(1)

Implementation:

Can use array or linked list

Real-Life Examples:

  • Printer queue
  • Customer service line
  • Task scheduling
  • Breadth-first search

Stack vs Queue

Stack (LIFO):

  • Like a stack of plates - take from top
  • Operations: push, pop, peek
  • Use when: order matters (undo, recursion)

Queue (FIFO):

  • Like a line at store - first person first served
  • Operations: enqueue, dequeue, peek
  • Use when: fairness matters (printing, scheduling)

Key Takeaways

  • Arrays: Fast access O(1), fixed size
  • Linked Lists: Dynamic size, O(n) access
  • Stacks: LIFO - last in, first out
  • Queues: FIFO - first in, first out
  • Choose based on your needs (access patterns, size changes)
  • Use built-in libraries in production
  • Understanding fundamentals helps debug and optimize
