Gewichten, heuristics en kortste paden

Samenvatting

Een weighted graph heeft kosten of gewichten op edges. Daardoor zoek je niet alleen een pad, maar het goedkoopste of kortste pad.

Heuristics zijn snelle vuistregels, maar garanderen niet altijd de optimale oplossing.

Dijkstra berekent de kortste afstand vanaf één startnode naar andere nodes, zolang de gewichten niet negatief zijn.

Wat moet je begrijpen?

  • Weight: kost op een edge.
  • Distance array: beste bekende afstand.
  • Visited: node waarvan kortste afstand vastligt.
  • Relaxation: afstand verbeteren via een buur.
  • Predecessor array: nodig om pad te reconstrueren.

Big O & complexiteit

  • Eenvoudige Dijkstra met matrix/scannen: O(V²).
  • Dijkstra met priority queue: O((V + E) log V).
  • BFS voor unweighted shortest path: O(V + E).

C# voorbeeld

// Idee van relaxation:
if (dist[u] != int.MaxValue && dist[u] + weight < dist[v])
{
    dist[v] = dist[u] + weight;
    previous[v] = u;
}

Typische valkuilen

  • Dijkstra gebruiken met negatieve gewichten.
  • Alleen afstand berekenen maar predecessor vergeten als je het pad moet tonen.
  • De lokale goedkoopste edge nemen en denken dat dit altijd Dijkstra is.
  • BFS gebruiken voor weighted shortest path.
